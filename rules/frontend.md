# Frontend

Read the @frontend/README.md for project-specific context.

Read @frontend/Makefile to learn which commands are available for frontend development and testing. Most common commands are:

* `make dist` - compile the frontend app
* `make check` - run static analysis on changed files
* `make unit` - run unit tests
* `make run` - starts two processes:
  * - **elm-land server**: Development server on `http://localhost:1234` with hot reloading. It proxies API requests to `http://localhost:8080` (the backend).
  * - **tailwindcss watch**: Watches Elm files and regenerates CSS.


## Folder Structure

```
frontend/
├── src/
│   ├── Pages/          # Elm Land pages (routes)
│   ├── Api/            # Auto-generated API client (DO NOT EDIT)
│   ├── style.css       # Tailwind CSS input
│   └── View.elm        # Custom view type (auto-generated by Elm Land)
├── static/             # Static assets (logo, generated CSS)
├── elm-land.json       # Elm Land configuration
├── elm.json            # Elm dependencies
├── process-compose.yml # Development server configuration
└── tailwind.config.js  # Tailwind CSS configuration
```

### `src/Pages/`

Elm Land uses file-based routing. Each `.elm` file in `src/Pages/` becomes a route:
- `Home_.elm` → `/`
- `Settings.elm` → `/settings`
- `Users/Id_.elm` → `/users/:id`

See [Elm Land routing docs](https://elm.land/guide/routing) for more info.

### `src/Api/`

Auto-generated API client from the backend's `openapi.yaml` spec. DO NOT EDIT - changes are overwritten whenever nix recreates the development environment.

## Code Style

- Use `elm-format` for all Elm code.
